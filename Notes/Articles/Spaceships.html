Spaceships: Battle for the Frontier

Introduction

Many Computer Science programs have students creating a large scale applicaiton at the end of their program. For me, that was Spaceships: Battle for the Frontier.

The project (a team based activity, which posed it's own difficulties) was to build a game using any language and any libraries we chose. This article outlines the details of the project.

The Game

Spaceships: Battle for the Frontier is a real time, online multiplayer, strategy game. Players are matched up in real time with other players from around the world. Each player has an assortment of 7 ships (2 Destroyers, 2 MineLayers, 2 Cruisers, 1 Gunboat) The game begins with the players selecting where they would like to place their ships. Once all the ships have been placed, the action begins!

Players must control their ships, each with varying speeds and capabilities, and attack and destroy either all the opponents ships, or their base. If a players ship is damaged, it can retreat back to the base in order to heal itself. Moreover, proximity mines can be laid that upon notification that a ship (enemy or friendly) is next to a mine, detonates damaging all ships in the surrounding area.

The Technical Details

Firstly, it should be noted that the source code is available, open source, at Github. Feel free to take a look.

The progam was written entirely in Java using the LibGDX development environment. We wrote the networking ourselves, using Java Sockets in order to transfer data. Moreover, we wrote the code for the server, that handled the majority of the game logic. This meant that players computers simply needed to have access to the graphics rendering code, meaning their own overhead was very low. Since the game was rendered in 2D using simple Geometric Shapes, the overhead on any one computer was quite low. This meant that the bottleneck, if any, would occur on the server, meaning both players would be affected equally, thus limiting the implications on gameplay.

If anyone wants to learn about LibGDX game development, I strongly urge them to check out Learning LibGDX Game Development by Andreas Oehike. This is the book we chose to use and it definitely made our lives easier.
